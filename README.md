# HakoniwaTimeSpec

Formal specification support for Hakoniwa time synchronization.

## Lean proof check

This repository contains a first Lean ideal model under `lean/`.

```bash
mix deps.get
mix test
```

`mix test` starts a LeanLsp runtime and runs:

```bash
cd lean
lake build HakoniwaTimeSpec
```

The Lean ideal model currently proves the initial bounded-drift invariant,
invariant preservation for asset and core steps, pairwise asset skew bounds for
reachable states, and the general no-deadlock condition. The Lean code avoids
`sorry`, `admit`, and version-fragile arithmetic automation in this first model.

The runtime is selected by `HAKONIWA_TIME_SPEC_LEAN_RUNTIME`:

- `local` (default): run `lake` on the host through `LeanLsp.Runtime.Local`.
- `docker`: run `lake` in Docker through `LeanLsp.Runtime.Docker`.
- `auto`: use a local `lake` executable if found, otherwise use Docker.

For Docker runs, the default image is `leanprovercommunity/lean4:latest`.
Override it with `HAKONIWA_TIME_SPEC_LEAN_DOCKER_IMAGE` when a pinned image is required.


## Guides

The generated documentation includes a guide for the first Lean model:

- `docs/integer-tick-ideal-model.md`: an English explanation of the integer-tick ideal model, with Lean excerpts and KaTeX equations.

## Documentation math rendering

`mix docs` is configured to load KaTeX for HTML documentation generated by ExDoc.
Use inline TeX with `$...$` and display equations with `$$...$$` in Markdown docs
and module documentation.

For example:

```markdown
The invariant is $T_i \le T_c \le T_i + D_{\max}$.

$$
\forall i,j,\; |T_i - T_j| \le D_{\max}
$$
```

KaTeX is injected only for the HTML formatter; EPUB generation keeps the extra
head and body tags empty.

## Installation

If available in Hex, the package can be installed by adding `hakoniwa_time_spec` to your list of dependencies in `mix.exs`:

```elixir
def deps do
  [
    {:hakoniwa_time_spec, "~> 0.1.0"}
  ]
end
```

Documentation can be generated with ExDoc and published on HexDocs.
